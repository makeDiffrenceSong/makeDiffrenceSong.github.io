trace分析
Created2020-11-23|Updated2020-12-01
|Post View:
“main” prio=5 tid=1 Native
线程名: main prio: 线程优先级 tid: 线程内部id 线程状态: NATIVE
group=”main” sCount=1 dsCount=0 flags=1 obj=0x73e11b38 self=0x7ebc614c00
线程所属的线程组 线程挂起次数 用于调试的线程挂起次数 当前线程关联的java线程对象 当前线程地址
sysTid=26053 nice=-10 cgrp=default sched=0/0 handle=0x7f42033548
线程真正意义上的tid 调度有优先级 进程所属的进程调度组 调度策略 函数处理地址
state=S schedstat=( 455228046 191645585 857 ) utm=36 stm=9 core=1 HZ=100
线程状态 CPU调度时间统计, 见proc/[pid]/task/[tid]/schedstat 用户态/内核态的CPU时间(单位是jiffies), 见proc/[pid]/task/[tid]/stat 该线程的最后运行所在核 时钟频率
stack=0x7fcdbca000-0x7fcdbcc000 stackSize=8MB
线程栈的地址区间 栈的大小
